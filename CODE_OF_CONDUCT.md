# Code of Conduct

## Our Standard

Contributors are expected to keep collaboration direct, respectful, and technically grounded.

Examples of expected behavior:

- discuss technical disagreements in concrete terms
- focus criticism on code, design, and evidence
- respect different experience levels and backgrounds
- accept correction without escalation
- keep discussions relevant to the work

Examples of unacceptable behavior:

- harassment, abuse, or intimidation
- personal attacks
- discriminatory language or conduct
- deliberate disruption of discussions or reviews
- publishing private information without permission

## Enforcement

Project maintainers are responsible for interpreting and enforcing this policy.

Maintainers may remove, edit, or reject comments, issues, pull requests, or other contributions that violate this code of conduct.

## Reporting

If you experience or witness unacceptable behavior, report it privately to the project maintainer through the repository owner contact path for this project.

## Scope

This code of conduct applies to repository discussions, issues, pull requests, reviews, and other project-managed collaboration spaces.
